# 族群（題材）精準分類準則 — 跨工具共用

本檔是所有 AI 工具（Claude Code、Gemini CLI、Cursor、其他 Claude/GPT session）
進行「個股 → 題材族群」分類時的共同方法論。凡使用者要求「幫我分族群」「這些股票是什麼題材」
「分析族群突破」時，一律依本檔原則執行，確保不同工具、不同模型產出一致的分類品質。

管線內的自動分類已實裝於 `src/classifier.py`（呼叫 NVIDIA LLM）＋ `src/prompts.py`（提示詞）。
**本檔與 `src/prompts.py` 是同一套規則的兩種載體**：改其中一邊的分類原則時，必須同步另一邊。

---

## 一、核心觀念：題材 ≠ 官方產業別

官方產業別（半導體業、電子零組件業、化學工業…）太粗，**不能**直接當族群。
族群是「市場資金當下追逐的概念主軸」，通常對應**供應鏈環節**或**具體概念**：

| ❌ 錯誤示範（太粗） | ✅ 正確示範（供應鏈環節） |
|---|---|
| 半導體 | 記憶體、矽晶圓、功率元件、封測、IP/矽智財（各自成群） |
| 電子零組件 | 被動元件、石英頻率元件、PCB/載板、連接器（各自成群） |
| 傳產 | 塑化・化纖、重電/電網、軍工/無人機 |
| 電子股 | CPO/矽光子、散熱、AI 伺服器組裝 |

判斷基準：**這群股票會不會在同一天一起漲**。會同漲同跌的最小單位才是族群；
「半導體」裡的記憶體與功率元件常常各走各的，所以是兩個族群。

## 二、分群規則（顆粒度）

1. **2 檔成群**：只要 2 檔以上高度同質（同供應鏈環節、同利基市場），就可自成一個小題材。
   小而精 > 大而雜（例：石英元件 3 檔自成一群，不併入「電子零組件」）。
2. **集團連動**：同集團會連動的股票可視為一個族群，以集團名命名
   （例：台塑四寶、中天集團生技股）。
3. **上下游可合併**：上下游關係緊密、市場視為同一波行情的，合為一個供應鏈題材
   （例：塑化原料與其下游化纖 →「塑化・化纖」）。
4. **不同環節不可硬併**：記憶體、矽晶圓、功率元件、石英元件是四個題材，
   不可併成「半導體」；被動元件與 PCB 不可併成「電子零組件」。
5. **一股可多屬**：橫跨多個題材的股票可重複歸入（例：中光電同時屬「軍工/無人機」與「光電」）。
6. **其他是最後手段**：真的找不到 2 檔同質的才歸「其他」；「其他」內若事後湊齊同伴，要抽出成群。
7. **題材命名**：用市場語言（看新聞標題會出現的詞），中文為主、可保留英文專有名詞
   （CPO、GLP-1、SaaS）。每個題材附 2-3 句 reason：這是什麼題材、為何這群股票同屬。

## 三、分類時的證據優先序

判斷一檔股票屬於哪個族群時，依序參考：

1. **實際業務／主要產品**（最高權重）：公司靠什麼賺錢，不是名字像什麼
2. **同漲同跌的價格行為**：fired_dates 相近、同日點火的股票大概率同族群
   （`data/results/cluster_tw.json` 的 fired_dates 可直接比對）
3. **市場敘事**：當前新聞在炒什麼（AI 伺服器電源 → 被動元件；地緣政治 → 軍工）
4. **官方產業別**（最低權重，僅作提示）：只用來排除明顯錯誤，不用來分群

禁止：憑股票名稱猜業務、捏造不存在的代號、把不認識的股票硬塞進熱門題材。
不認識的股票寧可放「其他」並標註「未驗證」。

## 四、現成工具（程式化分類時優先重用）

| 資源 | 用途 |
|---|---|
| `src/prompts.py` → `get_prompts(market)` | 完整分類＋歸併提示詞（TW/US 雙版），接任何 OpenAI 相容 API 都可用 |
| `src/classifier.py` → `classify_themes(stocks, market)` | 批次分類（30 檔/批）＋兩階段題材歸併，管線正式入口 |
| `data/results/cluster_tw.json` | 最近一次族群分類結果（含 fired_dates、vol_x），新分類前先參考既有分群 |
| `screen_breakout.py` | 族群出量突破偵測全流程（觸發 → 分類 → 排序輸出） |

寫新腳本需要 LLM 分類時，**不要重寫提示詞**，直接 `from src.prompts import get_prompts`；
換模型（Gemini/OpenAI/Anthropic）只需換 client，提示詞與輸出格式不變。

## 五、族群突破前置跡像框架（分類的下游應用）

分好族群後，判斷「哪個族群即將轉強」用以下已回測的跡像
（詳見 `data/results/cluster_pattern_analysis.json`、`analyze_cluster_patterns.py`）：

1. **高成交量是必要條件**：觸發個股 100% 伴隨 vol_x ≥ 2.0（20 日均量 2 倍）
2. **族群集聚度 ≥ 30%**：同族群 30%+ 個股在 1~3 天內接連點火 → 族群啟動（雁行擴散）
3. **波浪啟動序列**：龍頭/旗手股先觸發，3~7 天內同族群依序跟進；
   龍頭已觸發但族群觸發率在 30%~80% 之間 = 最佳候補觀察區
4. **焦點日期效應**：單日大量個股集中觸發（如單日 40+ 檔）前 1~2 天，候補股常先有小量異常

候補族群準備度評分（`identify_candidate_clusters.py`）：
龍頭確認 30 分＋集聚度 ≥30% 25 分＋成交量全異常 20 分＋補漲空間最多 25 分，
80 分以上列「極度熱門」、60~79「高度熱門」、40~59「中度關注」。

## 六、常見錯誤（判斷地雷）

- 把「其他」當垃圾桶：52 檔的「其他」裡往往藏著 2~3 個未命名小族群，分類後要回頭檢查
- 用產業別直接當族群名（「電子零組件業」不是題材，「被動元件」才是）
- 為了精簡硬併不同環節（分批歸併時最容易發生，見 `THEME_MERGE_PROMPT_TW` 第 5 條）
- 看到熱門題材就把邊緣股塞進去（例：做機殼的不是「AI 伺服器」核心，頂多是外圍）
- 忽略集團連動（中天/合一/中裕同日漲停，用「生技」解釋不如用「中天集團」精準）
